# 기본기 되짚기

이 저장소는 **Java와 Spring Boot의 기본기를 차근차근 되짚어보기 위한 학습 기록 공간**입니다.

프로젝트를 진행하면서 ‘왜 이렇게 되는지’, ‘이게 정확히 뭘 의미하는지’를 놓쳤던 부분들을

하나씩 다시 정리하고, 직접 코드로 검증한 결과를 남깁니다.

<br/>

## 🧭 항상 “왜?”를 생각하기

> 면접 준비를 하면서 나는 오랫동안 ‘정답’을 외우는 데에 집중했다.  
> 질문이 나오면 그에 맞는 문장을 빠르게 말하는 연습만 반복했다.  
> 하지만 시간이 지날수록, 그렇게 외운 지식은 금방 잊히고  
> ‘왜 이런 기술이 필요한가’를 이해하지 못한 채 겉만 따라가고 있었다.
>
> 그래서 이제는 어떤 기술을 배울 때마다 항상 **“왜?”** 를 먼저 묻도록 한다.

### 1️⃣ 무엇이 불편했기에 등장했는가?

- 기존 방식의 한계나 불편함을 찾는 단계

  - ex1) Spring Framework → “매번 객체 생성, 의존성 연결을 직접 해야 해서 유지보수가 어려움”
  
  - ex2) JPA → “SQL을 직접 작성하기 번거롭고, 객체-관계 불일치가 발생함”

### 2️⃣ 왜 이런 기술이 필요했는가?

- 기술의 ‘핵심 가치’를 짚는 단계

    - ex1) Spring은 “IoC / DI”를 통해 객체 생명주기와 의존성을 자동 관리.
  
    - ex2) JPA는 "객체 중심으로 개발"할 수 있도록 하여, 직접 SQL에 의존하지 않게 해준다.

### 3️⃣ 그 기술이 구체적으로 무엇을 해결했는가?

- 내부 구조와 작동 방식을 이해하는 단계

  - ex1) Spring → BeanContainer가 의존성 주입을 담당, AOP로 횡단 관심 분리
  
  - ex2) JPA → EntityManager가 1차 캐시, 트랜잭션 단위로 flush, dirty checking 수행
